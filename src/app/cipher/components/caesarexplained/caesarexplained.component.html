How it works

A Caesar Cipher is a very basic encryption method, the complexity and security of which can vary. At the most basic level, three things are required:

  1. A message to encrypt
  2. At least one key, usually a number
  3. A list of characters (letters, numbers, punctuation marks, special characters)

Encrypting

If using one single key, to encrypt a message, you would:

  1. Go through each character in the message
  2. Find that character in the list of characters
  3. Shift to the left or right by the number of the key
  4. Add that character to the new message
  5. Repeat for each character in the plain message
  6. If the key goes beyond the end of the character list, wrap around to the beginning
  7. Characters not in the list can be handled in different ways

Example, let's use this cipher on a simple message of 'secret message', and the following list of characters 'abcdefghijklmnopqrstuvwxyz ' (note that I included a space), and a key of 2 (keeping things simple for explaining) and shifting to the right to encrypt.

  • 's' becomes 'u'
  • 'e' becomes 'g'
  • 'c' becomes 'e'
  • 'r' becomes 't'
  • 'e' becomes 'g'
  • 't' becomes 'v'
  • ' ' becomes 'b'
  • 'm' becomes 'o'
  • 'e' becomes 'g'
  • 's' becomes 'u'
  • 's' becomes 'u'
  • 'a' becomes 'c'
  • 'g' becomes 'i'
  • 'e' becomes 'g'

'secret message' becomes 'ugetgvboguucig'

Decrypting

Decrypting a Caesar Cipher with a single key (if you know what the key is) is a matter of shifting each character the opposite direction (I used right above, so to decrypt, you would shift to the left). That's also why a single key Caesar Cipher is NOT secure, with a list of common words, a computer program could break this cipher very quickly.

Improving

There are ways to improve on the Caesar Cipher but enough to guard very important information so tread carefully.

  • Use multiple keys. Here are a few ways of encrypting using 3 keys:
    • Rotating Keys
      • The first character is shifted using the first key
      • The second is shifted using the second key
      • The third is shifted using the third key
      • The fourth is shifted using the first key
      • The fifth is shifted using the second key
      • The sixth is shifted using the third key
      • This method is used with the 2 Key Encryption above
    • Multiple Shifts
      • Every character would be shifted by the first key
      • Every second character would be shifted by the second key
      • Every third character would be shifted by the third key
      • These would overlap each other so some characters would be shifted as many as three times
    • Rotating Shift Direction
      • Shift to the right with the first key
      • Shift to the left with the second key
      • Back to the right with the third key
    • Combination of the ideas above or other possibilities
  • Using a rotating character list. Instead of using a-z all the time, randomize the order of the letters so they aren't always in alphabetical order. Randomizing just a-z lower case where no letter repeats, means there are 403 septillion possibilities. Now expand that to include upper case letters, numbers, punctuation marks and other characters. A different randomized set could be used each hour of each day for 1,000 years and barely scratch the number of possible sets. You would just need to make sure that both sides use the same character list...

There are many options available, you just need to make sure that the decryption method is the opposite of the encryption method.
